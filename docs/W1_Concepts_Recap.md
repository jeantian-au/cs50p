##  List Comprehension

### 1. 核心作用与方括号 `[]`
- **打包机制**：方括号 `[]` 在推导式中的作用是告诉 Python 解释器：“我要把里面循环出来的数据，打包组合成一个全新的列表（List）”。
- **嵌套结构（以矩阵为例）**：
    ```python
    new_matrix = [[row[i] for row in matrix] for i in range(4)]
    ```
    - **内层 `[]`**：纵向抓取每个小列表的第 `i` 位元素，打包成一个“新行”。
    - **外层 `[]`**：把所有做好的“新行”集装箱，打包成最终的大矩阵。

### 2. `if` 条件在 `for` 前后的巨大区别
- **放在 `for` 后面 ➡️ 筛选 (Filtering)**：
    - **语法**：`[x for x in data if x % 2 == 0]`
    - **特点**：新列表通常会**变小**。只有满足 `if` 条件的元素才会被保留。
- **放在 `for` 前面 ➡️ 转换 (Transforming)**：
    - **语法**：`[ 'even' if x % 2 == 0 else 'odd' for x in data ]`
    - **特点**：新列表与原列表**大小完全相同**。它必须配合 `else` 使用（三元运算符），对每个元素根据条件进行不同的转换。

### 3. 动态化范围 (Dynamic Range)
- 为了避免硬编码（如固定写死 `range(4)`），应当使用 `len()` 动态获取规整矩阵的列数：
    ``` python
    # 自动适应任意列数的规整矩阵
    [[row[i] for row in matrix] for i in range(len(matrix[0]))]
    ```

##  def main()
Python 脚本的执行机制
![[Pasted image 20260530172652.png]]
### 1. 定义 (Definition) vs 执行 (Execution)
- **两阶段工作流**：
    1. **定义阶段**：Python 自上而下扫描文件，遇到 `def` 时只是把函数的代码“蓝图”或“机器”组装好存入内存，**并不运行内部代码**。因此，函数的定义顺序无所谓。
    2. **执行阶段**：只有当代码流真正走到调用行（如 `main()`）时，才会去内存中找蓝图并启动。
- **铁律**：【模块顶层的调用】必须在定义之后。 + 但函数内部互相调用没有顺序要求(因为执行调用时所有 def 已扫描过)。

### 2. 标准入口：`if __name__ == "__main__":`
- **内置常量**：`"__main__"` 是 Python 解释器硬编码的特殊常量，**绝对不能**因为你写了 `def haha():` 就改成 `if __name__ == "__haha__":`。
- **核心场景区分**：
    - **直接运行脚本**：Python 自动将变量 `__name__` 赋值为 `"__main__"`，条件成立，执行主逻辑。
    - **作为模块被 import 导入**：`__name__` 会被赋值为它的文件名（如 `"script"`），条件不成立，内部代码被跳过。这保证了模块的干净和安全。
- **规范写法**：
    ```python
    def haha():
        print("我的核心逻辑")
    
    if __name__ == "__main__":
        haha()  # 固定的门牌号后面，调用你自定义的任何函数
    ```

##  Dictionary [ ] or .get()
在访问未知或可选的键（Key）时，取值方式决定了程序的稳定性：
### 中括号 `[]` 访问**：`fruits["mango"]`
    - **特点**：严格、直接。
    - **后果**：如果键不存在，程序会直接抛出 **`KeyError` 异常并崩溃**。适用于确定键一定存在的场景。
    -
### get()` 方法访问**：`fruits.get("mango", "默认值")`
    - **特点**：安全、灵活。
    - **后果**：如果键不存在，**绝不报错**。默认返回 `None`，或者返回你自定义的第二个参数（如 `"默认值"`）。

## Library vs List vs Dictionary

| **特性**   | **库 (Library) —— 工具箱**           | **列表 (List) —— 储物箱**              |
| -------- | -------------------------------- | --------------------------------- |
| **本质功能** | 提供预写好的复杂功能（模块、函数、类）              | 内置的、有序且可变的数据容器                    |
| **可变性**  | 属于固定实体，无法动态对其增删代码                | 高度可变，支持 `.append()` 或 `.remove()` |
| **使用前提** | 必须使用 `import` 语句导入               | 属于内置原生类型，直接用 `[]` 创建              |
| **实际生活** | 比如导入 `math` 算平方根，导入 `pandas` 算数据 | 比如存储宠物的名字：`["Dami", "Xiaomi"]`    |
### 三方对照（避免再混淆）

| 类型          | 是什么    | 创建方式      | 例子            |
| ----------- | ------ | --------- | ------------- |
| **List**    | 有序数据集合 | 直接 [] 创建  | `[1, 2, 3]`   |
| **Dict**    | 键值数据集合 | 直接 {} 创建  | `{"a": 1}`    |
| **Library** | 外部代码模块 | 必须 import | `import math` |
**前两者是【数据结构】(放数据用),后者是【代码工具】(放函数用)。**
**Dict 和 List 是亲兄弟,Library 和它们没关系。**

## Tuple
- **核心结论**：在 Python 中，**逗号 `,` 才是创建元组的灵魂，圆括号 `()` 只是起到了分组和整洁的作用**。
- **打包黑科技**：
    ``` python
    y = 1, 2  # 隐式元组打包
    print(type(y))  # 输出: <class 'tuple'>
    ```
- **单元素陷阱**：如果你想创建一个只有一个元素的元组，写成 `a = (5)` 最终会被识别为整数 `int`（括号被当做数学括号忽略了）。必须加上逗号：`a = (5,)`。

## zip(* matrix)
### 1. 它在干什么？
它是 **矩阵转置（行变列、列变行）** 的一行流终极快捷方式，包含两步：
1. **`*matrix`（星号解包）**：把大集装箱里的多行小列表“轰”地一声攤开，变成独立的散装参数。
2. **`zip()`（重组机器）**：纵向对齐，把所有参数相同索引位置的元素重新打包成一个个新元组。
### 2. 致命误区：`zip` 返回的是什么？
- `zip()` 返回的 **绝对不是 List**，而是一个 **`zip` 对象（迭代器/生成器）**。
- 它具有 **Lazy Evaluation（惰性计算）** 的特性，内存中只有一个指针，不包含真实实体数据。
- 因此：zip(*pets)[0] 或 zip(*pets)(0) 都会触发 TypeError（运行时错误,不是语法错误）。 + 错误信息：'zip' object is not subscriptable 

### 3. 正确的过滤与提取列
想拿转置后的某一列，必须先用 `list()` 集装箱把散装货物接住，让它转化为实体集合，再用中括号索引：
``` python
# 提取第一列 (索引 0)
first_col = list(zip(*pets))[0]

# 提取不连续的多列 (例如第 0 和第 3 列，配合推导式)
transposed_list = list(zip(*pets))
filtered_result = [transposed_list[i] for i in [0, 3]]
```
### 4. 局限性：木桶效应 (Unequal Lengths)
- `zip()` 的并行处理满足木桶效应，**一旦最短的那个子列表结束，机器立刻停止运行**。
- **前提条件**：使用 `zip(*matrix)` 的矩阵必须是**非常规整的**（行长列宽完全对应）。如果数据不规整，会导致长列表中的尾部数据被静默丢弃（数据丢失）。
- **实际生活用处**：常用于快速将两个分离的列表缝合成字典（`dict(zip(names, scores))`），或者在物联网中同步读取多个并行传感器

## List & Dictionary 解析
### 1. 添加操作 (ADD)
* **列表 (List):**
* `my_list.append('date')`：在列表末尾追加元素。
* `my_list.insert(1, 'blueberry')`：在指定索引位置（此处为索引 1）插入元素。
* **字典 (Dict):**
* `my_dict['date'] = 4`：直接通过新键赋值。
* `my_dict.update({'date': 4})`：使用字典批量更新/添加。

### 2. 删除操作 (DELETE)
* **列表 (List):**
* `del my_list[1]`：删除指定索引处的元素。
* `my_list.remove('banana')`：删除列表中第一个匹配的指定值。
* `my_list.pop()`：移除并返回列表末尾的元素。
* **字典 (Dict):**
* `del my_dict['banana']`：删除指定的键值对。
* `my_dict.pop('banana')`：删除指定的键并返回其对应的值。
* `my_dict.popitem()`：删除并返回最后插入的键值对（元组形式）。

### 3. 更新操作 (UPDATE)
* **列表 (List):**
* `my_list[0] = 'kiwi'`：通过索引直接修改该位置的值。
* **字典 (Dict):**
* `my_dict['apple'] = 10`：通过已存在的键直接修改对应的值。
* `my_dict.update({'apple': 10})`：通过 `update` 方法修改值。

## List & Dictionary return value 返回值函数与方法详解

返回值是指方法执行完毕后交付给程序的数据。以下是高频使用的相关函数：
### 1. 通用函数
* **`len(collection)`**
* **列表:** 返回元素的总个数（整数）。
* **字典:** 返回键值对的总个数（整数）。

### 2. 字典特有的“视图对象”方法
这些方法返回特殊的只读视图对象（随原字典动态更新），通常用于循环遍历：
* **`my_dict.keys()`**
* **返回:** 包含所有键的视图（`dict_keys(['apple', 'cherry', 'date'])`）。
* **`my_dict.values()`**
* **返回:** 包含所有值的视图（`dict_values([10, 3, 4])`）。
* **`my_dict.items()`**
* **返回:** 包含所有键值对元组的视图（`dict_items([('apple', 10), ('cherry', 3), ('date', 4)])`）。
### 3. 字典的安全获取方法：`.get()`
为了防止直接用 `my_dict[key]` 访问不存在的键时导致程序崩溃（报 `KeyError`），推荐使用 `.get()`：
* `my_dict.get('cherry')` $\rightarrow$ 键存在，返回对应的值 `3`。
* `my_dict.get('banana')` $\rightarrow$ 键不存在，默认安全返回 `None`。
* `my_dict.get('banana', 0)` $\rightarrow$ 键不存在时，返回你指定的默认兜底值 `0`。

## dict() 的三种用法

除了使用花括号 `{}` 外，内置的 `dict()` 构造函数提供了更灵活的字典创建方式：
1. **从另一个映射/字典创建（复制）**
```python
source_dict = {'a': 1, 'b': 2}
my_dict1 = dict(source_dict)  # 创建了一个新副本

```
2. **从键值对的可迭代对象（如元组列表）创建**
```python
kv_pairs = [('x', 10), ('y', 20), ('z', 30)]
my_dict2 = dict(kv_pairs)  # 自动转化为 {'x': 10, 'y': 20, 'z': 30}

```
3. **从关键字参数创建**
```python
# 注意：关键字参数的键不能加引号，必须是符合 Python 变量命名规范的标识符
my_dict3 = dict(name='Alice', age=28)  # 自动转化为 {'name': 'Alice', 'age': 28}

```
